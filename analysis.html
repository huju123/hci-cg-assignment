Norman’s Design Principles

Don Norman emphasizes that good design supports human cognition through visibility, feedback, mapping, affordances, and constraints.

  • Visibility: Navigation links are always visible.
  • Feedback: Correct and incorrect HTML examples provide immediate learning feedback.
  • Mapping: Labels are placed directly above form controls.
  • Constraints: Users are guided using structured navigation.
  • Affordance: Buttons and links visually suggest interaction.

These principles reduce the gulfs of execution and evaluation by making actions and system states clear.

GOMS Model

GOMS analyzes user interaction in terms of Goals, Operators, Methods, and Selection rules.

  • Goal: Learn semantic HTML concepts.
  • Operators: Click, scroll, read.
  • Method: Navigate pages → study examples → practice errors.
  • Selection Rule: Beginners start from Home, advanced users jump to Errors or Quiz.

This structure supports efficient task completion for expert, error-free interaction.

Suchman’s Situated Action Theory

Lucy Suchman argues that human actions are not strictly planned but emerge from interaction with the situation. Users adapt dynamically based on feedback from the system.

This website supports situated action by allowing free exploration, providing error recovery examples, and enabling users to learn through interaction rather than forcing a fixed learning path.

Application Summary

  • Norman: Clear navigation and feedback support usability.
  • GOMS: Tasks are structured logically.
  • Suchman: Users learn through exploration and correction.
